Overview

Recovery Calendar is a Progressive Web App (PWA) that helps users track recovery meetings, find nearby meetings, and manage their recovery schedule. It works like a native app — installable on phones, works offline, and syncs across devices.

The entire front-end lives in a single index.html file. The back-end consists of Netlify serverless functions and a separate Meeting Aggregator API that crawls and indexes meetings from national databases.

Architecture

User's Browser (PWA) | +-- index.html (all UI, JS, CSS) +-- sw.js (service worker / offline) +-- manifest.json (PWA install) | +-- Firebase (auth + realtime DB) +-- EmailJS (email notifications) | +-- Netlify Functions | +-- find-meetings.js (legacy search) | +-- parse-schedule.js (schedule parser) | +-- chat.js (AI chat helper) | +-- Meeting Aggregator API (meeting-aggregator.netlify.app) +-- /api/search (ZIP + radius + filters) +-- /api/admin/crawl (trigger data refresh) +-- Neon Postgres (56,000+ meetings) +-- BMLT feeds (NA — 51 state queries) +-- 95 AA intergroup feeds (~40 states)

Meeting Data Sources

🌐 BMLT (Narcotics Anonymous) ~23,000 meetings

The Basic Meeting List Toolbox is the standard API for NA meetings worldwide. The aggregator queries the "Tomato" server (a global aggregator) once per US state to get complete national coverage. Returns structured JSON with meeting names, times, locations, and coordinates.

Parser: bmlt_json (51 state queries)

BMLT.app ↗ | Tomato Server ↗
△ Meeting Guide API (Alcoholics Anonymous) ~33,000 meetings

AA has no single national feed. Each intergroup runs a WordPress site with the "12 Step Meeting List" (TSML) plugin, which exposes a JSON API. The aggregator pulls from 95 intergroup feeds covering approximately 40 US states, providing near-national AA coverage.

Parser: meeting_guide
To add more AA coverage: Find an intergroup website, check if /wp-admin/admin-ajax.php?action=meetings returns JSON, and add it to prisma/seed.ts.

PWA Features

📷 Service Worker sw.js

Enables offline functionality and caching. The service worker caches the app shell so the PWA works even without internet. Located at sw.js in the project root.

📱 Web App Manifest manifest.json

Defines how the app appears when installed on a phone: name, icons, theme color, and display mode. Located at manifest.json in the project root.

Netlify Serverless Functions

Three serverless functions run on Netlify, defined in netlify/functions/:

find-meetings.js — Legacy meeting search (fallback if aggregator is unavailable)
parse-schedule.js — Parses meeting schedule text/images into structured data
chat.js — AI chat helper for recovery-related questions

To manually trigger a refresh (the .env file already contains the Neon DATABASE_URL):

cd ~/Claude\ Projects/Recovery\ Calendar/meeting-aggregator
npm run dev
Then in a separate terminal tab (Cmd+T):
curl -X POST http://localhost:3000/api/admin/crawl -H "Content-Type: application/json" -d '{"deduplicate":false}'
The crawl takes 15-30 minutes to process all 95+ feeds.
➕ Adding New AA Coverage

To add a new AA intergroup to the aggregator:

1. Find an AA intergroup website that uses the "12 Step Meeting List" WordPress plugin
2. Test if their feed works: visit [site]/wp-admin/admin-ajax.php?action=meetings
3. If it returns JSON, add the source to prisma/seed.ts
4. Run npm run db:seed then npm run crawl (with DATABASE_URL for cloud)

Search API Reference

🔍 POST /api/search

The main search endpoint. Accepts JSON body with these parameters:

zipCode: string ......... 5-digit US ZIP code
radiusKm: number ........ Search radius in km (default: 16)
limit: number ........... Max results (default: 200)
program: string[] ....... ["AA", "NA", "SMART", "Al-Anon"]
day: number[] ........... [0=Sun, 1=Mon, ... 6=Sat]
format: string[] ........ ["Discussion", "Speaker", etc.]
access: string .......... "Open" or "Closed"
attendanceMode: string .. "In-Person", "Virtual", "Hybrid"
textSearch: string ...... Search meeting names
city: string ............ Filter by city name
state: string ........... Filter by 2-letter state code

Response includes: meetingName, program, dayOfWeek, startTime, endTime, format, access, attendanceMode, locationName, street, city, state, zip, lat, lng, virtualUrl, virtualMeetingId, virtualPassword, confidenceScore, distanceKm, distanceMiles

🌐 Meeting Attendance Modes
In-Person — Physical meeting at a location with street address
Virtual — Online meeting via Zoom, Google Meet, etc. Includes virtualUrl, virtualMeetingId, and virtualPassword when available
Hybrid — Both in-person and virtual options available

Virtual/Hybrid meetings without coordinates are included in radius searches without distance filtering, so they always appear in results regardless of location.
