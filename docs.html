Authentication & Data
🔥 Firebase Auth + Realtime Database

Handles user authentication (Google sign-in and email/password) and stores each user's meeting data in Firebase Realtime Database. Data syncs in real-time across devices. The app also works offline via localStorage with sync-on-reconnect.

Meeting Aggregator
🔍 Meeting Aggregator API Next.js + Prisma + TypeScript

A separate Next.js application that crawls meeting data from national sources, normalizes and deduplicates it, and serves it via a search API. The PWA calls this when users search by ZIP code in "Find Meetings."

Tech: Next.js 14, TypeScript, Prisma ORM, PostgreSQL
Live: meeting-aggregator.netlify.app
Endpoint: POST /api/search

Meeting Data Sources
🌐 BMLT (Narcotics Anonymous) ~23,000 meetings

The Basic Meeting List Toolbox is the standard API for NA meetings worldwide. The aggregator queries the "Tomato" server (a global aggregator) per US state for complete national coverage. Returns structured JSON with meeting names, times, locations, coordinates, and virtual meeting links.

Parser: bmlt_json (51 state queries)

BMLT.app ↗ Tomato Server ↗
△ Meeting Guide API (Alcoholics Anonymous) ~33,000 meetings

AA has no single national API. Each local intergroup runs a WordPress site with the "12 Step Meeting List" (TSML) plugin, which exposes a JSON feed. The aggregator pulls from 95 intergroup feeds covering approximately 40 US states.

Parser: meeting_guide
Feed pattern: [site]/wp-admin/admin-ajax.php?action=meetings

Search API Reference
🔌 POST /api/search

The main search endpoint. Accepts JSON body:

zipCode: string ......... 5-digit US ZIP code radiusKm: number ........ Search radius in km (default: 16) limit: number ........... Max results (default: 200) program: string[] ....... ["AA", "NA", "SMART", "Al-Anon"] day: number[] ........... [0=Sun, 1=Mon, ... 6=Sat] format: string[] ........ ["Discussion", "Speaker", etc.] access: string .......... "Open" or "Closed" attendanceMode: string .. "In-Person", "Virtual", "Hybrid" textSearch: string ...... Search meeting names city: string ............ Filter by city name state: string ........... Filter by 2-letter state code

Response fields: meetingName, program, dayOfWeek, startTime, endTime, format, access, attendanceMode, locationName, street, city, state, zip, lat, lng, virtualUrl, virtualMeetingId, virtualPassword, confidenceScore, distanceKm, distanceMiles

🌐 Meeting Attendance Modes

In-Person — Physical meeting at a location with street address
Virtual — Online via Zoom, Google Meet, etc. Includes virtualUrl, virtualMeetingId, and virtualPassword
Hybrid — Both in-person and virtual options available

Virtual/Hybrid meetings without coordinates are included in radius searches without distance filtering.

PWA Features
📷 Service Worker sw.js

Enables offline functionality and caching. The service worker caches the app shell so the PWA works even without internet.

📱 Web App Manifest manifest.json

Defines how the app appears when installed: name, icons, theme color, and standalone display mode. Installable on iOS, Android, and desktop.

Netlify Serverless Functions

Three serverless functions in netlify/functions/:

find-meetings.js — Legacy meeting search (fallback if aggregator unavailable)
parse-schedule.js — AI-powered parsing of meeting schedule text/images into structured data
chat.js — AI chat helper for recovery-related questions
